3. Installation

Follow these steps to set up the workspace:

# 1. Create a new ROS 2 workspace (if you don't have one)
mkdir -p ~/ros2_ws/src
cd ~/ros2_ws/

# 2. Clone the repository into your workspace's src directory
git clone https://github.com/RAS598-2025-S-Team01/project_code.git src/project_code

# 3. Install dependencies (ensure ROS 2 Humble and other prerequisites are installed)
sudo apt update
rosdep update
rosdep install --from-paths src --ignore-src -r -y
# Install any pip dependencies if needed
# pip install -r src/project_code/[path_to_requirements.txt]

# 4. Build the workspace
colcon build --symlink-install

# 5. Source the workspace
source install/setup.bash
# (Optional: Add this line to your ~/.bashrc for persistence)

4. Usage

To launch the system:

# Source your workspace if you haven't already
source ~/ros2_ws/install/setup.bash

# Launch the main system components using the provided launch file
# (Assuming you have a launch file named main_launch.py in one of your packages)
ros2 launch [PLACEHOLDER: package_name] [PLACEHOLDER: main_launch.py]

After launching, you should be able to:

  • Access the web UI by navigating to http://[IP_ADDRESS_OF_HOST]:[PORT] in your browser (e.g., http://localhost:5000 if running locally).
  • View ROS 2 topics, nodes, and services using standard ROS 2 command-line tools (ros2 topic list, ros2 node list, etc.).
  • Control the quadruped and observe sensor data via the web interface.
  • Trigger the autonomous workflow (details depend on your specific implementation - may involve UI buttons or specific ROS 2 service calls).
